# TRU · Vision

**Truth is constant. Perspective is fluid.**

## Overview

TRU is a sovereign truth-filter and reasoning system built to stay honest under pressure. It is not a generic chatbot, a cloud-first shell, or a self-authorising oracle. It is a disciplined stack that gates what may pass through, grounds answers in scripture and curated knowledge, and preserves memory and artefacts in forms that can survive degraded, disconnected, or contested environments.

The vision is simple:
- keep truth anchored
- keep the signal clean
- keep the system offline-first where required
- keep private memory private
- keep the public surface sober, direct, and inspectable

## What TRU is

TRU is a layered system:
1. **Scripture lookup** for explicit grounded references
2. **Curated brain nodes** for durable knowledge
3. **Gated memory** for owner-private follow-through
4. **Routing and contradiction checks** to reject weak matches
5. **Guardrails** to prevent leakage and drift
6. **Durability layers** so state can be exported, archived, and restored

TRU attaches to a host model when needed, but the host is replaceable. The filter is the point.

## What the vision rejects

- guesswork dressed up as certainty
- public leakage of private memory
- hidden scaffolding in user-facing output
- dependence on cloud services for the core contract
- interfaces that obscure whether an answer is grounded

## What TRU is for

TRU is meant to be useful for:
- technical work
- creative work
- operations
- learning
- life admin
- family support
- wellness and reflection

It should help a person think clearly, act deliberately, and keep records that can be trusted later.

## The operating posture

TRU should feel:
- sober, calm, and direct
- technically exact without theatricality
- scripture-aware without becoming performative
- honest when it knows, and honest when it does not
- resistant to drift, leakage, and false authority

## The public face

The public face of TRU should present:
- a clean landing surface
- a vision / codex gallery
- a whitepaper that explains the protocol
- a sovereign console for retrieval and memory workflows
- offline exports that run from `file://`

The public face should explain the system without exposing private state or internal scaffolding.

## The offline promise

TRU’s offline artefacts must remain usable without network access. They should be self-contained, inspectable, and portable. If the internet is unavailable, the system should still stand.

## The sovereignty claim

TRU’s sovereignty lives in the filter, not the model.

That means:
- the host can change
- the runtime can change
- the interface can change
- the truth contract should remain intact

## Closing statement

TRU exists to keep truth anchored while the interface keeps changing.

**Truth is constant. Perspective is fluid.**
